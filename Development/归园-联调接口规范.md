## 1. 接口约定

### 1.1 基础信息
- **协议**: HTTPS
- **数据格式**: JSON
- **字符编码**: UTF-8
- **时间格式**: ISO 8601
- **金额单位**: C币为整数，人民币为元（保留2位小数）

### 1.2 请求头
```
Content-Type: application/json
Authorization: Bearer {jwt_token}
X-Request-ID: {uuid}
```

### 1.3 响应格式
```typescript
// 成功
{ "code": 0, "message": "success", "data": {} }

// 失败
{ "code": 1001, "message": "错误信息", "data": null }
```

### 1.4 错误码

| 码段 | 说明 |
|-----|------|
| 0 | 成功 |
| 1xx | 通用错误 |
| 2xx | 认证相关 |
| 4xx | 地块相关 |
| 5xx | 订单相关 |
| 6xx | 支付相关 |
| 7xx | C币相关 |
| 8xx | 托管相关 |

**具体错误码：**

| 错误码 | 说明 |
|-------|------|
| 0 | success |
| 1001 | 参数错误 |
| 1002 | 服务器内部错误 |
| 2001 | token过期 |
| 2002 | token无效 |
| 4001 | 地块不存在 |
| 4002 | 地块已售出 |
| 5001 | 订单不存在 |
| 5002 | 订单已支付 |
| 6001 | 支付失败 |
| 7001 | C币余额不足 |
| 8001 | 托管套餐不存在 |

## 4. 开发顺序建议

1. **第一阶段**：登录 + 首页 + 地块列表/详情
2. **第二阶段**：C币充值 + 地块购买
3. **第三阶段**：装饰编辑器 + 种植选择
4. **第四阶段**：托管服务 + 生长日志
5. **第五阶段**：订单中心 + 个人中心
6. **第六阶段**：优化 + 测试 + 联调
